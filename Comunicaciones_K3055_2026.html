    📶 Señales y Transmisión

    Señales Analógicas vs Digitales

    • Analógica: Toma infinitos valores continuos. La información está en la "forma de la onda". Ejemplo: voz, audio.
    • Digital: Toma un número finito de valores (ej: 0 y 1). La información está en la "codificación" (secuencia binaria).
    • MODEM: MODulador/DEModulador — convierte digital→analógico para redes analógicas.
    • CODEC: COdificador/DECodificador — convierte analógico→digital para redes digitales.
    • Repetidor regenerativo: Recrea la señal digital limpia, eliminando ruido acumulado. Ventaja clave sobre el amplificador analógico.

    Problemas de transmisión

    • Atenuación: Pérdida de energía de la señal con la distancia.
    • Ruido: Señales no deseadas que se mezclan con la señal útil.
    • Distorsión: Cambio en la forma de la onda durante la transmisión.
    • Retardo (Delay): Tiempo de propagación + procesamiento.
    • Jitter: Variación de retardo en conmutación de paquetes.
    ⚖️ OSI vs TCP/IP
    Modelo OSI (7 capas)
    Modelo de referencia teórico de ISO/UIT-T. Universal pero no el estándar de facto en Internet.
    Modelo TCP/IP (4 capas)
    Estándar de facto de Internet. Desarrollado por DARPA → base de toda Internet actual.
    Aplicación (capas 5,6,7 OSI)
    Transporte → TCP / UDP
    Internet → IP
    Acceso a la red (capas 1,2 OSI)
    🌐 Topologías de Red
    • Estrella: Nodo central concentra todo el tráfico. Ej: LAN con switch.
    • Malla: Todos conectados entre sí. Alta disponibilidad. Sin nodo central.
    • Anillo: Cada nodo conectado a los dos adyacentes. Hoy: redes industriales / ópticos.
    • Bus/Barra: Todos al mismo canal. Requiere protocolo de acceso. Ej: Ethernet 802.3.
    • Híbrida: Combinación de las anteriores para superar limitaciones.
    🔀 Tipos de Conmutación
    • Circuitos: Camino dedicado extremo a extremo. Recursos fijos. Latencia constante. Ej: telefonía analógica.
    • Mensajes: Mensaje completo se almacena y reenvía en cada nodo.
    • Paquetes: Información dividida en paquetes independientes. Rutas variables. Puede haber jitter, pérdida y reordenamiento. Sólo funciona con señales digitales. Ej: Internet.
    🏛️ Organismos de Normalización
    ISO / IEC
    Define el modelo OSI (ISO/IEC 7498-1). Seguridad: ISO 27001.
    ITU-T / UIT
    Telecom internacional. G.652 (fibra óptica), H.264/265 (video), G.711 (voz).
    IEEE
    802.3 Ethernet · 802.11 Wi-Fi · 802.1Q VLAN · Gigabit Ethernet.
    IETF
    Protocolos de Internet mediante RFC. RFC 791 (IP), 793 (TCP), 7540 (HTTP/2).
    ICANN / IANA
    Coordinan DNS global y asignación de direcciones IP. Delegación a RIR regionales (LACNIC para Latinoamérica).
    TIA / EIA
    Cableado estructurado: TIA/EIA-568 (Ethernet). RS-485 (TIA-485).
    CLASE 1 27/03/2026 · Presencial · OSI, Telecomunicaciones, Internet

    1. Introducción a Telecomunicaciones

    • Telecomunicaciones: Comunicación a distancia que supera la presencia física. Tele (lejano) + communicatio.
    • Evolución: Telégrafo → Telefonía → Televisión → Transmisión de datos → Internet → IoT → 5G + AI.
    • Teleinformática: Convergencia de telecomunicaciones e informática. Permite procesamiento distribuido y servicios de valor agregado.
    • Ley de Moore: El número de transistores en un chip se duplica ~cada 2 años. Efecto parcialmente mantenido hoy con integración 3D y chiplets.
    • Clasificación de redes por alcance: LAN (local), MAN (metropolitana), WAN (amplia).

    2. Modelo OSI — Conceptos Clave

    • 7 capas: Física, Enlace, Red, Transporte, Sesión, Presentación, Aplicación.
    • Comunicación horizontal (peer-to-peer entre capas iguales) mediante protocolos. Comunicación vertical mediante primitivas.
    • Encapsulamiento: Cada capa agrega un header al PDU recibido de la capa superior. Física transmite bits; Enlace tramas; Red paquetes; Transporte segmentos.
    • Dispositivos por capa: Hub/Repeater (capa 1) · Switch/Bridge (capa 2) · Router (capa 3).
    • Un socket = dirección IP + número de puerto + protocolo de transporte. Ej: 192.168.1.10:80/TCP.

    3. Internet

    • Red de redes independientes interconectadas con protocolo IP (estándar de facto: TCP/IP).
    • Historia: ARPANET 1969 → TCP/IP 1974 → WWW 1990 → uso comercial masivo 1995.
    • DNS: Sistema jerárquico de nombres de dominio. 13 root servers (A a M). ICANN coordina. IANA asigna IPs globales. LACNIC para Latinoamérica.
    • IPv4: 32 bits (ej: 192.168.0.1). IPv6: 128 bits (ej: 2001:0db8:...). Protocolo IP permite conmutación y enrutamiento.
    • Nuevas tecnologías: FTTH, GPON, MPLS, SDH-NG, DWDM.
    CLASE 2 31/03/2026 · Virtual · Redes, Señales, Conmutación

    4. Comunicación de Datos

    • Estructura física: ETD (Equipo Terminal de Datos) ↔ ECD (Equipo de Comunicaciones de Datos) ↔ canal.
    • Requisitos: confiable, sin errores, en tiempo real, flexible, segura.
    • En red analógica se usa MODEM. En red digital se usa CODEC (COdificador/DECodificador).
    • Ventajas de la transmisión digital: mayor alcance, calidad ajustable, integración con PCs, empleo de fibra óptica, costos más económicos.
    • Repetidor regenerativo (digital): recrea la señal limpia — elimina ruido. Vs amplificador (analógico): amplifica también el ruido.

    5. Elementos y Topologías de Red

    • Nodo: punto con 2+ vínculos y facilidades de conmutación. Vínculo: medio físico entre nodos. Terminal: equipo que genera/extrae señales.
    • Sin conmutación, links necesarios para n nodos = n(n-1)/2. Los conmutadores reducen dramáticamente la cantidad de enlaces.
    • Topología estrella: nodo central concentra tráfico. Punto único de fallo. Ej: LAN con switch.
    • Topología malla: todos conectados entre sí. Alta disponibilidad. Para sistemas críticos.
    • Topología anillo: cada nodo a sus adyacentes. Hoy: redes industriales y anillos ópticos.
    • Topología bus: todos al mismo canal. Requiere protocolo de acceso. Ej: 802.3/Ethernet clásico.
    • Híbrida: combinación de anteriores. Industria: estrella/bus para sensores con PLC, malla entre PLCs.

    6. Conmutación

    • De circuitos: camino dedicado E2E antes de transmitir. 3 fases: establecimiento → transferencia → desconexión. Latencia constante. Ej: telefonía.
    • De mensajes: cada mensaje completo se almacena en un nodo y se reenvía.
    • De paquetes: info dividida en paquetes independientes. Rutas variables. Puede haber jitter, pérdida y reordenamiento. Solo funciona con señales digitales.
    • Funciones básicas de una red: Conmutación + Transmisión + Señalización.
